# 第 1 階段筆記：後端基礎

## 重要概念

### API 是什麼？
API 是前端和後端溝通的「窗口」。
前端（網頁）發送請求 → 後端處理 → 回傳資料給前端。

### HTTP 請求類型
- `GET` — 取得資料（查詢）
- `POST` — 新增資料
- `DELETE` — 刪除資料

### Endpoint（端點）
一個 API 的網址，例如 `/api/bookmarks`，不同網址對應不同功能。

### JSON
後端回傳資料的格式：
```json
[{ "id": 1, "title": "Google", "url": "https://google.com" }]
```

## 程式碼觀念

### MapGet / MapPost / MapDelete
- `Map` = 登記一個端點，把網址和程式碼綁在一起
- 後面的 `Get` / `Post` / `Delete` = 指定接受哪種 HTTP 方法
- 兩邊要對得上，後端才會處理請求

### Lambda（匿名函式）
沒有名字、直接傳入的函式，用 `=>` 表示：
```cs
app.MapPost("/api/bookmarks", (BookmarkInput input, AppDbContext db) =>
{
    // 這整段程式碼是「傳進 MapPost 的第二個參數」
});
```
等同於先定義函式再傳入，只是寫法更簡潔。

### Class 的用途
- 定義資料的「形狀」，確保每個物件都有一致的欄位
- 整個專案都認識這個形狀，可以在函式參數、清單、資料庫等地方使用
- 強型別：填錯欄位或型別，編譯器立刻報錯

### 為什麼有兩個 Class（Bookmark 和 BookmarkInput）
- `Bookmark`：資料庫裡的完整資料（含 Id、CreatedAt）
- `BookmarkInput`：前端傳來的資料（只有 Title、Url、Note）
- `Id` 和 `CreatedAt` 由後端自動產生，不讓前端指定

### 依賴注入（DI）
只需要在程式啟動時登記一次，之後任何地方在參數寫上型別，.NET 自動幫你建立並傳入：
```cs
// 登記
builder.Services.AddDbContext<AppDbContext>(...);

// 使用（.NET 自動傳入 db，不需要自己 new）
app.MapGet("/api/bookmarks", (AppDbContext db) => { ... });
```

### async / await
資料庫查詢需要時間，用 `async/await` 讓程式在等待時不會卡住：
```cs
// 等待資料庫回應，但不阻塞其他請求
return await db.Bookmarks.ToListAsync();
```
