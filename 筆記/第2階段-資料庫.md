# 第 2 階段筆記：資料庫

## 重要概念

### Code First vs Database First
- **Database First**：先在資料庫手動建資料表，再讓程式讀取
- **Code First（我們用的）**：先在 C# 寫好 Model，EF Core 自動建資料表

Code First 的好處：
- 改欄位只要改 C#，不用手動操作資料庫
- 資料表結構變更可以用 git 追蹤
- 部署到新環境只要跑一行指令

### Entity Framework Core（EF Core）
C# 的 ORM 工具，讓你用 C# 操作資料庫，不需要寫 SQL。

### DbContext（AppDbContext）
代表「資料庫的連線」，同時定義資料庫裡有哪些資料表：
```cs
public class AppDbContext : DbContext
{
    public DbSet<Bookmark> Bookmarks { get; set; }  // 對應一張資料表
}
```

### DbSet
代表資料庫裡的一張資料表，可以對它進行 CRUD 操作：
```cs
db.Bookmarks.ToListAsync()   // 查詢全部
db.Bookmarks.FindAsync(id)   // 用主鍵查詢
db.Bookmarks.Add(bookmark)   // 新增（還沒寫入）
db.Bookmarks.Remove(bookmark) // 刪除（還沒寫入）
db.SaveChangesAsync()         // 真正寫入資料庫
```

### Add 和 SaveChanges 是兩個步驟
- `Add()` / `Remove()` — 只是告訴 EF Core「我要做這件事」
- `SaveChangesAsync()` — 才真正送出 SQL 到資料庫
- 好處：可以一次 Add 很多筆，最後一起 SaveChanges，效能較好

## Migration 指令

```bash
# 安裝 EF Core 指令工具
dotnet tool install --global dotnet-ef

# 產生 Migration 檔案（把 C# Model 翻譯成 SQL 建表語法）
dotnet ef migrations add 名稱

# 套用到資料庫（真正建立資料表）
dotnet ef database update
```

---

## 資料表關聯
EF Core 支援資料表之間的關聯，只要在 C# Model 裡定義，Migration 會自動產生對應的外鍵：
```cs
public class Bookmark
{
    public int UserId { get; set; }   // 外鍵欄位
    public User User { get; set; }    // 導覽屬性（告訴 EF Core 關聯關係）
}
```
