# 專案檔案結構說明

## 整體結構

```
c:\code learning\
├── backend\      ← 後端（C# .NET）
├── frontend\     ← 前端（Next.js）
└── 筆記\         ← 學習筆記
```

---

## 前後端的根本差異

| | 後端（.cs） | 前端（.tsx/.ts） |
|--|------------|----------------|
| 執行位置 | 伺服器上 | 使用者的瀏覽器裡 |
| 能做什麼 | 存取資料庫、處理邏輯 | 顯示畫面、處理使用者互動 |
| 使用者看得到嗎 | 看不到 | 看得到（F12） |

兩者不直接溝通，透過 **HTTP 請求（fetch）** 交換資料。

---

## 副檔名說明

| 副檔名 | 語言 | 用途 |
|--------|------|------|
| `.cs` | C# | 後端邏輯 |
| `.tsx` | TypeScript + JSX | 前端（有畫面的檔案） |
| `.ts` | TypeScript | 前端（純邏輯，無畫面） |
| `.json` | JSON | 設定檔 |

`.tsx` 和 `.ts` 的差別：`.tsx` 可以寫 JSX（HTML 語法），`.ts` 不行。

## backend\ 檔案說明

### 需要動的檔案
| 檔案 | 作用 |
|------|------|
| `Program.cs` | 程式進入點，定義所有 API 端點、設定資料庫和 CORS |
| `Models/Bookmark.cs` | 書籤的資料結構，EF Core 依此建立資料表 |
| `Data/AppDbContext.cs` | 告訴 EF Core 資料庫裡有哪些資料表 |
| `appsettings.json` | 設定檔，存放資料庫連線字串 |
| `test.http` | API 測試檔，用 REST Client 發送請求 |

### 不需要手動修改的
| 檔案/資料夾 | 作用 |
|-------------|------|
| `appsettings.Development.json` | 開發環境設定，會覆蓋 appsettings.json |
| `Properties/launchSettings.json` | 控制啟動設定（port 號等） |
| `backend.csproj` | 專案設定，記錄套件和 .NET 版本 |
| `Migrations/` | EF Core 自動產生，記錄資料庫結構變更歷史 |
| `bin/` | 編譯後的執行檔，自動產生，不要動 |
| `obj/` | 編譯暫存檔，自動產生，不要動 |

---

## frontend\ 檔案說明

### 需要動的檔案
| 檔案 | 作用 |
|------|------|
| `app/page.tsx` | 首頁（對應網址 /），書籤功能的主畫面 |
| `app/layout.tsx` | 所有頁面共用的外框，改這裡影響整個網站 |
| `app/globals.css` | 全域 CSS 樣式 |
| `next.config.ts` | Next.js 設定，部署 Azure 時會用到 |

### 不需要手動修改的
| 檔案 | 作用 |
|------|------|
| `package.json` | 專案設定，記錄套件和可執行指令 |
| `package-lock.json` | 自動產生，記錄套件精確版本，不要動 |
| `tsconfig.json` | TypeScript 編譯設定 |
| `eslint.config.mjs` | 程式碼風格檢查設定 |
| `postcss.config.mjs` | Tailwind CSS 處理設定 |
| `next-env.d.ts` | 自動產生，不要動 |
| `.gitignore` | 告訴 git 不需要追蹤哪些檔案 |
| `public/` | 靜態檔案（圖片等），可直接用網址存取 |

## Next.js 網址對應規則

資料夾名稱就是網址，每個資料夾裡放一個 `page.tsx`：

```
app/page.tsx                → 網址 /
app/about/page.tsx          → 網址 /about
app/bookmarks/page.tsx      → 網址 /bookmarks
app/bookmarks/[id]/page.tsx → 網址 /bookmarks/1, /bookmarks/2...
```

## package.json 的常用指令

```bash
npm run dev    # 啟動開發伺服器（localhost:3000）
npm run build  # 打包成正式版本（部署用）
npm run start  # 啟動正式版本
npm run lint   # 檢查程式碼風格
```

## 哪些資料夾要動、哪些不要動

| 資料夾 | 需要手動修改？ |
|--------|--------------|
| `backend/Models/` | 要，資料結構定義 |
| `backend/Data/` | 偶爾，新增資料表時 |
| `frontend/app/` | 要，所有頁面都在這裡 |
| `backend/bin/` | 絕對不要 |
| `backend/obj/` | 絕對不要 |
| `backend/Migrations/` | 不要手動改，讓指令自動產生 |
| `frontend/node_modules/` | 絕對不要 |
| `frontend/.next/` | 絕對不要 |
