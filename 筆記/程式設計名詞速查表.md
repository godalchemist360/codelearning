# 程式設計名詞速查表

---

## 基本概念

### 變數（Variable）
存放資料的容器，給它一個名字方便之後使用。
```cs
var name = "Google";  // name 是變數，裡面存著 "Google"
var count = 5;        // count 是變數，裡面存著 5
```

---

### 型別（Type）
告訴程式「這個變數裡面存的是什麼種類的資料」。
```cs
int    count = 5;          // 整數
string name = "Google";    // 文字
bool   isActive = true;    // 是/否（true/false）
double price = 9.99;       // 小數
DateTime time = DateTime.Now; // 日期時間
```

---

### 函式 / 方法（Function / Method）
一段有名字的程式碼，可以重複呼叫。
- 獨立存在的叫「函式」
- 屬於某個物件/類別的叫「方法」
- 實務上兩個詞常混用
```cs
// 定義一個函式
int 加法(int a, int b)
{
    return a + b;
}

// 呼叫它
var result = 加法(3, 5); // result = 8
```

---

### 參數（Parameter）與引數（Argument）
- **參數**：定義函式時寫的佔位符號
- **引數**：實際呼叫時傳入的值
```cs
// a 和 b 是「參數」
int 加法(int a, int b) { return a + b; }

// 3 和 5 是「引數」
加法(3, 5);
```
> 日常對話中這兩個詞常混用，不用太糾結

---

### 回傳值（Return Value）
函式執行完後，交出來的結果。
```cs
int 加法(int a, int b)
{
    return a + b;  // 把結果交出去
}

var result = 加法(3, 5);  // result 接住回傳值 8
```
如果函式不需要回傳任何東西，型別寫 `void`。

---

### 類別（Class）
定義一種「資料的形狀 + 它能做的事」的模板。
```cs
public class Bookmark        // 定義模板
{
    public string Title { get; set; }  // 屬性
    public string Url { get; set; }    // 屬性
}
```

---

### 物件（Object）
用 `new` 從 Class 建立出來的實體，每個物件都是獨立的。
```cs
var a = new Bookmark { Title = "Google", Url = "https://google.com" };
var b = new Bookmark { Title = "GitHub", Url = "https://github.com" };
// a 和 b 是兩個獨立的物件
```

---

### 屬性（Property）
物件擁有的資料欄位。
```cs
bookmark.Title  // 存取 bookmark 的 Title 屬性
bookmark.Url    // 存取 bookmark 的 Url 屬性
```

---

### 實例化（Instantiation）
用 `new` 把 Class「變成」物件的動作。
```cs
var bookmark = new Bookmark();  // 把 Bookmark 這個模板實例化
```

---

## 進階概念

### Lambda（匿名函式）
沒有名字、直接寫在某個地方的函式，用 `=>` 表示。
```cs
// 一般函式
int 加法(int a, int b) { return a + b; }

// Lambda（同樣功能，但沒有名字）
(int a, int b) => a + b
```

---

### 非同步（Async / Await）
當某個操作需要等待（例如查資料庫、呼叫 API），
用 `async/await` 讓程式在等待時不會卡住。
```cs
// 沒有 async：等資料庫時整個程式卡住
var list = db.Bookmarks.ToList();

// 有 async：等資料庫時可以先處理其他事
var list = await db.Bookmarks.ToListAsync();
```

---

### 介面（Interface）
只定義「應該有什麼方法」，不寫實作內容。
像是一份合約，規定某個 Class 必須提供哪些功能。
（目前專案暫時不會用到，知道有這個東西就好）

---

### 依賴注入（Dependency Injection, DI）
讓 .NET 自動幫你建立和管理物件，不需要自己 `new`。
```cs
// 先登記
builder.Services.AddDbContext<AppDbContext>(...);

// 之後任何地方要用，直接寫在參數裡，.NET 自動傳入
app.MapGet("/api/bookmarks", (AppDbContext db) => { ... });
//                                         ↑ 不需要自己 new，.NET 自動給你
```

## Web / API 相關

### HTTP 請求方法
| 方法 | 用途 |
|------|------|
| GET | 取得資料 |
| POST | 新增資料 |
| PUT | 完整更新資料 |
| PATCH | 部分更新資料 |
| DELETE | 刪除資料 |

---

### 端點（Endpoint）
API 的一個網址，代表一個特定功能的入口。
```
GET    /api/bookmarks      ← 一個端點
POST   /api/bookmarks      ← 另一個端點
DELETE /api/bookmarks/1    ← 又另一個端點
```

---

### HTTP 狀態碼
伺服器回應時附帶的數字，告訴前端這次請求的結果。
| 狀態碼 | 意思 |
|--------|------|
| 200 OK | 成功 |
| 201 Created | 新增成功 |
| 204 No Content | 成功，但沒有資料要回傳（通常是刪除後） |
| 400 Bad Request | 前端傳來的資料有問題 |
| 404 Not Found | 找不到這個資源 |
| 500 Internal Server Error | 後端自己出錯了 |

---

### JSON
前後端傳遞資料的格式，像是一個結構化的文字。
```json
{
    "id": 1,
    "title": "Google",
    "url": "https://www.google.com",
    "note": "搜尋引擎"
}
```

---

### DTO（Data Transfer Object）
專門用來在層與層之間傳遞資料的 Class，
只包含資料，不包含邏輯。
我們的 `BookmarkInput` 就是一種 DTO。

---

## 資料庫相關

### ORM（Object-Relational Mapping）
讓你用程式語言（C#）操作資料庫，不需要直接寫 SQL。
Entity Framework Core 就是 C# 的 ORM 工具。

### Migration
記錄資料庫結構變更歷史的檔案，
可以用指令自動套用到資料庫。

### 主鍵（Primary Key）
資料表裡用來唯一識別每一筆資料的欄位，通常是 `Id`。

### 外鍵（Foreign Key）
指向另一張資料表主鍵的欄位，用來建立資料表之間的關聯。
